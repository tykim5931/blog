linux에서 fork()와 pthread_create()의 차이 fork()는 자기 자신과 똑같은 새 프로세스를 만드는 것이고, pthread()는 자신이 원하는 작업을 태스크의 형태로 생성한다. 따라서 엄밀히 말하면 pthread로 생성한 쓰레드가 진정한 쓰레드이고, fork로 생성한 것은 엄밀히 말하면 자식 프로세스로 보아야 한다. 결국 fork()에서 만든 새로운 프로세스는 fork()를 호출한 부모 프로세스의 메모리 공간에 접근할 수 없고, 자식 프로세스가 특정 전역변수의 값을 바꾸어도 부모 프로세스가 보고 있는 동명의 변수는 값이 변하지 않는다. 즉, 따로 수행시킨 두 프로그램과 큰 차이를 보이지 않는다. 반면 pthread()로 만든 쓰레드는 그렇지 않고, 두 쓰레드가 바꾼 전역변수 값이 모두 잘 보이게 된다 따라서 스레드 사이에서 데이터를 주고받는 방식은 아주 쉽다.
